## 1. 长视频 vs 短视频的本质区别

| 维度 | 短视频（1-3 分钟） | 长视频（5-30 分钟+） |
|------|-------------------|---------------------|
| 镜头数量 | 4-12 个 | 20-120+ 个 |
| 叙事结构 | 单一冲突线 | 多线并行（主线+副线） |
| 角色发展 | 简单弧光 | 完整角色弧光（信念转变） |
| 节奏控制 | 匀速推进 | 张弛有度（呼吸节奏） |
| 场景数量 | 1-3 个 | 5-15+ 个 |
| 核心挑战 | 讲好一个点 | 维持注意力 + 信息密度 |
| 生成策略 | 单轮生成 | 分段生成 + 一致性控制 |

### 长视频的核心公式

```
长视频 = 多幕结构 × 节奏曲线 × 角色弧光 × 视觉主题
```

**四个支柱缺一不可：**
- **多幕结构**：3 幕（5-10min）/ 4 幕（10-20min）/ 5 幕（20min+）
- **节奏曲线**：紧张/释放交替，不能一直高强度
- **角色弧光**：角色必须有信念/状态的转变
- **视觉主题**：统一的色调/构图语言贯穿全片

## 2. 剧本架构系统

### 2.1 多幕结构模板

#### 3 幕结构（5-10 分钟 / 20-40 镜头）

```
第一幕：建立世界（25%）━━━━━━━━━━━━━━━━━━━━━━━━
├─ 开场画面（1镜）：确立视觉基调
├─ 日常状态（2-4镜）：角色日常，建立共情
├─ 触发事件（1镜）：打破日常
└─ 决定出发（1镜）：角色做出选择

第二幕：对抗升级（50%）━━━━━━━━━━━━━━━━━━━━━━━━
├─ 初次尝试（3-5镜）：小胜利 + 新发现
├─ 中点转折（1镜）：⭐ 全片最重要的转折
├─ 深化危机（3-5镜）：代价越来越大
├─ 至暗时刻（1-2镜）：一切似乎完了
└─ 发现转机（1镜）：最后一丝希望

第三幕：高潮与解决（25%）━━━━━━━━━━━━━━━━━━━━━━━━
├─ 最终决战（3-5镜）：用上所有学到的
├─ 结局（1-2镜）：新的平衡
└─ 终场画面（1镜）：呼应开场，展现变化
```

#### 4 幕结构（10-20 分钟 / 40-80 镜头）

在 3 幕基础上，将第二幕拆分为 2A + 2B：

```
第一幕：建立（20%）→ 第二幕A：探索（30%）→ 第二幕B：对抗（30%）→ 第三幕：解决（20%）
```

### 2.2 节奏曲线设计

**呼吸节奏公式：**

```
紧张度：高 ━━ 低 ━━ 中 ━━ 高 ━━ 低 ━━ 极高
         ↑      ↑      ↑      ↑      ↑      ↑
        开场   喘息   积蓄   高潮   余韵   终场
```

**每 60-90 秒必须有一个节奏变化点：**
- 紧张 → 舒缓：角色独处、环境展示、内心独白
- 舒缓 → 紧张：新威胁出现、倒计时、意外发现
- 持续紧张：必须插入"信息揭示"来刷新注意力

### 2.3 角色弧光模板

```
角色弧光 = 初始信念 → 挑战 → 动摇 → 新信念

示例（成长型）：
初始："我一个人就够了"
挑战：独自面对失败
动摇：意识到需要帮助
新信念："我们一起可以做到"

示例（堕落型）：
初始："我要保护所有人"
挑战：保护所有人导致更大灾难
动摇：必须做出残酷选择
新信念："只牺牲少数才能拯救多数"
```

### 2.4 分镜脚本标准格式（长视频版）

每个镜头必须包含以下字段：

```yaml
镜头编号: S01-001  # Scene-编号
时长: 5s
幕: 第一幕
功能: 建立世界/展示角色/推进剧情/情绪转折
━━━━━━━━━━━━━━━━━━━━
景别: 全景/中景/近景/特写
角度: 平视/仰视/俯视/鸟瞰/过肩
运动: 固定/推/拉/摇/移/跟/升/降
━━━━━━━━━━━━━━━━━━━━
画面描述:
  主体: 谁/什么在画面中心
  动作: 正在发生什么（必须是可见的物理动作）
  环境: 周围有什么，天气/光线如何
  构图: 主体在画面什么位置
  色彩: 主色调，明暗对比
━━━━━━━━━━━━━━━━━━━━
情绪: 这一镜要让观众感受到什么
转场: 如何过渡到下一镜（切/淡入淡出/匹配剪辑/声音先导）
━━━━━━━━━━━━━━━━━━━━
模型: HappyHorse-t2v / Wan2.7-t2v / HappyHorse-i2v / ...
参考图: [需要哪些 reference 图]
━━━━━━━━━━━━━━━━━━━━
```

## 3. 模型适配 Prompt 工程

### 3.1 HappyHorse 1.0 Prompt 规范

**核心原则：** HappyHorse 对自然语言理解更强，适合描述性长 prompt。

#### HappyHorse 文生视频 (t2v)

```
[时间段描述] + [景别] + [主体] + [动作] + [环境] + [光影] + [情绪] + [风格]

GoodCase 示例：
"全景，缓慢推进。一个穿着褪色牛仔外套的中年男人独自走在荒凉的
戈壁滩上，夕阳把他的影子拉得很长。远处是连绵的沙丘轮廓，天空
被染成橘红色和深紫色的渐变。风卷起细小的沙尘在他脚边旋转。
他的步伐缓慢而坚定，偶尔停下来抬头看远方。电影质感，浅景深，
孤独而壮美的氛围。"
```

#### HappyHorse 图生视频 (i2v)

```
首帧图必须包含完整构图，Prompt 只需描述动作：

"一只橘猫从画面左侧慢慢走向右侧，在画面中央停下来，抬头看向
画面右上方，尾巴轻轻摆动。微风吹动它的毛发。"
```

#### HappyHorse 参考生视频 (r2v)

```
characterN 指代必须精确：

"身着红色旗袍的女性character1，镜头先以侧面中景勾勒旗袍修身
剪裁与S型曲线，随即切换至低角度仰拍，捕捉她轻抬玉手展开
折扇character2时流苏耳坠character3随头部转动轻盈摆动的细节，
最后推近至面部特写，定格在她指尖轻点扇骨、眼波流转间的含蓄风情。"

注意：
- character1 对应 media[0]，character2 对应 media[1]...
- 最多 9 张参考图
- 描述中必须用 characterN 指代，不能用"她/他/它"
```

### 3.2 Wan 2.7 Prompt 规范

**核心原则：** Wan 2.7 需要更结构化的 prompt，支持多镜头叙事。

#### Wan 2.7 文生视频 (t2v) — 多镜头叙事

```
第1个镜头[0-5秒] 全景：雨夜的纽约街头，霓虹灯在积水中倒映，
一个撑着黑伞的男人匆匆走过。
第2个镜头[5-10秒] 中景：男人进入一家老旧的咖啡馆，推门时风
铃响起，店内温暖的灯光照在他脸上。
第3个镜头[10-15秒] 近景：他坐在靠窗的位置，摘下帽子，露出
疲惫但坚定的眼神，窗外雨滴滑过玻璃。

风格：黑色电影，高对比度，蓝橙色调。
```

#### Wan 2.7 图生视频 (i2v) — 首尾帧

```
首帧：角色站在门口
尾帧：角色已走进房间深处
Prompt："他从门口走进房间，边走边环顾四周，最终停在房间中央。
房间内有温暖的灯光和复古的家具。"
```

### 3.3 模型选择决策树

```
需要生成什么？
│
├─ 纯文本 → 视频
│   ├─ 需要多镜头叙事？ → Wan 2.7 t2v
│   └─ 单镜头高质量？ → HappyHorse t2v
│
├─ 有参考图 → 视频
│   ├─ 只要首帧？ → HappyHorse i2v（宽高比自动跟随）
│   ├─ 需要首尾帧精确控制？ → Wan 2.7 i2v
│   ├─ 需要视频续写？ → Wan 2.7 i2v (--first-clip)
│   └─ 多角色融合（1-9张图）？ → HappyHorse r2v
│
├─ 已有视频需要编辑？
│   ├─ 风格转换/局部替换？ → HappyHorse video-edit（支持60s输入）
│   └─ 简单指令编辑？ → Wan 2.7 videoedit
│
└─ 需要音频驱动？
    └─ 只有 Wan 2.7 支持（t2v/i2v 均可加音频）
```

### 3.4 Prompt 禁忌清单

| ❌ 不要写 | ✅ 应该写 | 原因 |
|----------|----------|------|
| "他感到孤独" | "他独自坐在空荡的房间里，低头看着手机屏幕" | 情绪必须可视化 |
| "画面很美" | "金色的夕阳洒在湖面上，波光粼粼" | "美"太抽象 |
| "他们吵架了" | "她双手抱胸背对着他，他站在门口握紧拳头，茶几上摔碎的水杯" | 用动作和道具表达 |
| "突然" | "一扇门猛地被推开" | "突然"不是画面 |
| "回忆中" | 直接切到过去的场景，用色调区分 | 视频没有"回忆"这个视觉 |
| "他想了想" | "他停下来，眉头微皱，手指轻轻敲击桌面" | 思考不可见 |
| 一段 prompt 塞 3 个场景 | 拆成 3 个镜头分别生成 | 单段视频无法处理多场景跳跃 |

## 4. GoodCase 案例库

### 4.1 治愈系短旅（5 分钟 / 20 镜头）

**适合模型：** HappyHorse t2v（单镜头质量优先）

**风格：** 吉卜力动画风格，温暖色调

```markdown
# 《纸飞机的旅行》

时长: 5 分钟 | 镜头: 20 | 风格: 吉卜力手绘风

## 故事梗概
一个小女孩折的纸飞机被风吹出窗外，经历了一场城市冒险，最终
落回她的手中。

## 镜头 GoodCase

### 镜头 1 [8s] — 开场：折纸飞机
景别: 近景 → 特写
运动: 固定
画面: 一双小手在木质书桌上折叠彩纸。阳光从窗户斜射进来，照
在纸飞机上。桌上散落着蜡笔和贴纸。小女孩（8岁，扎着两个
丸子头，穿着黄色连衣裙）专注地折着最后一个折角，然后举起
纸飞机，对着阳光看了看。
情绪: 温暖、专注
色彩: 暖黄色调，阳光感
Prompt (HappyHorse):
"近景，缓慢推近至特写。一双小手在洒满阳光的木质书桌上折叠
彩色纸。阳光从画面右侧的窗户斜射进来，在纸飞机上投下温暖的
金色光芒。桌上散落着彩色蜡笔和星星贴纸。纸飞机是明亮的天蓝
色，折痕整齐。小手把纸飞机举到眼前，透过阳光仔细检查。吉卜
力手绘动画风格，温暖柔和的光线，充满童真的氛围。"
参考图: [女孩角色图, 书桌场景图]

### 镜头 5 [6s] — 中点：纸飞机飞过城市上空
景别: 全景
运动: 跟拍
画面: 纸飞机在城市上空飞行，下方是密集的屋顶、烟囱和小巷。
远处的电视塔和摩天大楼在天际线闪烁。一只鸽子从旁边飞过，
好奇地看了纸飞机一眼。
情绪: 自由、冒险
色彩: 蓝天，城市灰绿色调
Prompt (HappyHorse):
"全景，跟拍。一架天蓝色的纸飞机在东京风格的城市上空滑翔。
下方是密集的日式屋顶，瓦片在阳光下微微反光，小巷中隐约可见
行走的人们。远处的晴空塔和现代建筑群在天际线闪烁。一只灰色
的鸽子从画面左侧飞入，好奇地转头看向纸飞机，然后加速飞向前
方。吉卜力风格，蓝天背景，云层蓬松洁白，自由冒险的氛围。"

### 镜头 12 [5s] — 至暗时刻：纸飞机掉进水沟
景别: 近景
运动: 快速下摇 → 固定
画面: 纸飞机被风吹得失去平衡，旋转着掉进路边的排水沟。脏水
溅起，纸飞机在污水中漂着，一角已经浸湿。
情绪: 紧张、失落
色彩: 冷色调，阴暗
Prompt (HappyHorse):
"近景，镜头快速下摇。天蓝色纸飞机被一阵强风吹得失去平衡，
在空中翻滚两圈后落入路边的排水沟。污水溅起水花，纸飞机漂浮
在浑浊的水面上，一个角已经浸湿变软。沟壁长满青苔，水面漂浮
着落叶。吉卜力风格，冷灰色调，紧张失落的氛围。"

### 镜头 20 [10s] — 终场：呼应开场
景别: 全景 → 中景 → 特写
运动: 缓慢拉远
画面: 傍晚。小女孩坐在窗台上，手里拿着那架纸飞机（有些皱
但完整）。她把纸飞机放在窗台的花盆旁，旁边多了一排她新折
的纸飞机。夕阳把整个房间染成橘红色。
情绪: 释然、成长
色彩: 橘红→暖金渐变
Prompt (HappyHorse):
"全景，缓慢拉远至中景再到特写。傍晚时分，小女孩（丸子头、
黄色连衣裙）坐在窗台上，手中捧着那架有些褶皱但完整的天蓝色
纸飞机。她把纸飞机轻轻放在窗台的花盆旁边，旁边整齐排列着五
架新折的纸飞机，颜色各异。夕阳透过窗户，把整个房间染成温暖
的橘红色和金色，光线柔和。窗外可见城市天际线的剪影。吉卜力
手绘风格，温暖治愈，释然成长的氛围，浅景深。"
```

### 4.2 悬疑微电影（8 分钟 / 32 镜头）

**适合模型：** Wan 2.7 t2v（多镜头叙事能力更强）

**风格：** 大卫·芬奇风格，冷色调，高对比度

```markdown
# 《第 13 级台阶》

时长: 8 分钟 | 镜头: 32 | 风格: 悬疑惊悚

## 故事梗概
一个男人每天走同一段楼梯回家，今天他发现楼梯多了一级。
每次数都是 13 级。他开始怀疑，多出来的那级台阶通向哪里。

## 关键镜头 GoodCase

### 镜头 1 [5s] — 建立日常
Wan 2.7 Prompt (多镜头叙事):
"第1个镜头[0-2秒] 全景固定：一栋老旧的公寓楼入口，灰色的
水泥墙面，昏黄的路灯。一个穿着灰色风衣的男人走进画面。
第2个镜头[2-5秒] 中景跟拍：男人踏上楼梯，皮鞋踩在水泥台阶
上发出沉闷的回声。他熟练地数着台阶，嘴唇微动。冷色调，
高对比度，压抑的氛围。"

### 镜头 8 [5s] — 发现异常
Wan 2.7 Prompt:
"第1个镜头[0-2秒] 特写固定：男人的皮鞋踏上最后一级台阶。
第2个镜头[2-3秒] 近景快速推：男人突然停住，眉头紧皱。
第3个镜头[3-5秒] 特写缓慢下摇：他的视线落在脚下——台阶
比记忆中多了一级。冷蓝色调，不安的氛围，画面微微晃动。"

### 镜头 16 [8s] — 中点转折（关键）
Wan 2.7 Prompt:
"第1个镜头[0-3秒] 中景仰视缓慢推进：男人站在楼梯底部向上看。
楼梯在昏暗的灯光下无限延伸，每一级台阶都一模一样，看不到
尽头。第2个镜头[3-6秒] 特写固定：他的呼吸变得急促，额头冒
出细密的汗珠，手扶在冰冷的墙壁上。第3个镜头[6-8秒] 全景
缓慢旋转：楼梯开始扭曲变形，墙壁上的光影像水波一样晃动，
空间感开始崩塌。冷色调，高对比度，恐惧和眩晕的氛围。"
```

### 4.3 产品展示视频（3 分钟 / 12 镜头）

**适合模型：** HappyHorse r2v（产品一致性要求高）

## 5. 角色一致性管理

### 5.1 角色设定卡模板

```yaml
角色名: "小雅"
━━━━━━━━━━━━━━━━━━━━
外貌:
  年龄: 8岁
  发型: 两个丸子头，黑色短发
  服装: 黄色连衣裙，白色袜子，红色小皮鞋
  体型: 瘦小
  特征: 左脸颊有一颗小痣
━━━━━━━━━━━━━━━━━━━━
性格:
  关键词: 好奇、勇敢、有点莽撞
  小动作: 思考时咬下嘴唇
━━━━━━━━━━━━━━━━━━━━
视觉锚点 (每次必出现):
  - 黄色连衣裙
  - 丸子头
  - 红色小皮鞋
━━━━━━━━━━━━━━━━━━━━
参考图生成 Prompt:
  正面: "8岁小女孩正面全身照，黄色连衣裙，白色袜子，红色
    小皮鞋，两个黑色丸子头，左脸颊有小痣，白色背景，
    角色设定图风格"
  侧面: "同上，45度侧面"
  表情集: "同上，5种表情：开心/好奇/害怕/生气/思考"
━━━━━━━━━━━━━━━━━━━━
```

### 5.2 一致性保障策略

```
策略 1：参考图先行
  生成任何包含角色的镜头前，先成角色设定图
  → HappyHorse r2v / Wan 2.6 r2v-flash 使用角色参考图

策略 2：描述词锁定
  每个镜头的 prompt 中必须包含角色的完整外貌描述
  → 不能只写"小女孩"，必须写"穿黄色连衣裙扎丸子头的小女孩"

策略 3：首帧锁定
  同一角色在不同场景的镜头，使用同一张角色图作为首帧
  → HappyHorse i2v / Wan 2.7 i2v

策略 4：分段生成 + 后期筛选
  同一镜头生成 2-3 个版本，选择角色最一致的那个
  → 固定 seed 提高复现性
```

## 6. 完整生产工作流

### 6.1 阶段总览

```
Phase 1: 剧本创作 (1-2 小时)
  ↓ 输出：完整剧本 + 分镜脚本
Phase 2: 视觉开发 (2-4 小时)
  ↓ 输出：角色设定图 + 场景设定图 + 风格参考
Phase 3: 分镜生成 (4-8 小时，可并行)
  ↓ 输出：所有镜头的视频片段
Phase 4: 剪辑拼接 (1-2 小时)
  ↓ 输出：粗剪版完整视频
Phase 5: 音频 + 精剪 (2-3 小时)
  ↓ 输出：最终成片
```

### 6.2 Phase 1：剧本创作

```
输入: 主题/类型/时长/风格
  ↓
1. 确定核心冲突（一句话）
2. 选择幕结构（3幕/4幕/5幕）
3. 设计角色弧光
4. 编写分镜脚本（逐镜描述）
5. 标注每个镜头的生成模型
  ↓
输出: 完整分镜脚本（含 Prompt）
```

### 6.3 Phase 2：视觉开发

```
1. 生成角色设定图（正面/侧面/表情集）
   模型: wan2.7-image-pro 或 qwen-image-2.0-pro

2. 生成场景设定图（每个主场景一张）
   模型: wan2.7-image-pro

3. 生成关键道具图（需要在多镜头出现的道具）

4. 确认视觉风格一致性
  ↓
输出: 角色图 N 张 + 场景图 M 张 + 道具图 K 张
```

### 6.4 Phase 3：分镜生成

```
按场景分组，并行生成：

场景 A（5个镜头）:
  ├─ 镜头 A1: HappyHorse t2v "..."
  ├─ 镜头 A2: HappyHorse i2v --first 场景A设定图 "..."
  ├─ 镜头 A3: HappyHorse r2v --ref 角色图1 --ref 角色图2 "..."
  └─ ...

场景 B（8个镜头）:
  ├─ 镜头 B1: ...
  └─ ...

关键原则：
1. 每个镜头生成后立即检查角色一致性
2. 不一致则调整 prompt 或更换参考图
3. 重要镜头生成 2-3 个版本选最优
```

### 6.5 Phase 4：剪辑拼接

```bash
# 使用 ffmpeg 拼接所有镜头
ffmpeg -f concat -safe 0 -i filelist.txt -c copy output.mp4

# 添加转场效果
ffmpeg -i clip1.mp4 -i clip2.mp4 \
  -filter_complex "[0:v][1:v]xfade=transition=fade:duration=0.5:offset=4.5[outv]" \
  -map "[outv]" output.mp4

# 调整统一帧率和分辨率
ffmpeg -i input.mp4 -vf "fps=24,scale=1920:1080" output.mp4
```

### 6.6 Phase 5：音频 + 精剪

```
1. 添加背景音乐（BGM）
2. 添加音效（环境音、动作音）
3. 添加配音（TTS 生成）
4. 音量平衡
5. 最终导出
```

## 7. 常见问题与解决方案

### Q1: 角色在不同镜头长得不一样怎么办？

```
解决方案（按优先级）：
1. 使用 HappyHorse r2v，传入角色参考图
2. 在 prompt 中加入更详细的外貌描述
3. 使用同一张角色图作为所有镜头的首帧（i2v）
4. 固定 seed 值
5. 生成多个版本后选最一致的
```

### Q2: 长视频节奏太平怎么办？

```
检查节奏曲线：
1. 每 60-90 秒是否有节奏变化？
2. 是否有"信息揭示"刷新观众注意力？
3. 高潮前是否有足够的铺垫？
4. 是否有喘息镜头让观众消化信息？

调整方法：
- 在连续高强度镜头后插入 1-2 个舒缓镜头
- 在平淡段落加入意外事件或信息揭示
- 调整镜头时长（紧张段用短镜头，舒缓段用长镜头）
```

### Q3: 不同模型生成的镜头风格不统一怎么办？

```
1. 同一场景尽量使用同一模型
2. 在 prompt 中加入统一的风格描述（如"吉卜力手绘风格"）
3. 使用相同的参考图作为风格锚点
4. 后期调色统一色调
```

### Q4: 提示词太长被截断怎么办？

```
HappyHorse: 最长 5000 非中文字符 / 2500 中文字符
Wan 2.7: 最长 5000 字符

解决方案：
1. 优先保留：景别 + 主体 + 动作 + 环境
2. 精简形容词，保留核心视觉元素
3. 使用英文 prompt（同样长度包含更多信息）
```
